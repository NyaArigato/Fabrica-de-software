Sprint Track - Sistema de Acompanhamento de Equipes e Sprints

O Sprint Track é uma solução tecnológica desenvolvida para a gestão visual e o monitoramento ágil de fluxos de trabalho.
O projeto tem como objetivo acadêmico consolidar o rastreamento de tarefas e o progresso de iterações em uma interface centralizada, permitindo uma visão estratégica do desempenho da equipe.

O sistema resolve problemas como a falta de organização de tarefas, dificuldade no acompanhamento de prazos e a ausência de centralização de informações administrativas. Através dele, é possível gerenciar Sprints, atribuir responsáveis e visualizar o progresso por meio de um quadro Kanban e dashboards analíticos.

Tecnologias Utilizadas
Linguagem Principal: Java (JDK 17+)
Framework: Spring Boot (Back-end API) 
Banco de Dados: MySQL (Persistência de dados)
Front-end: HTML5, CSS3 e JavaScript (Interface do usuário) 
Arquitetura: Monolito Modular utilizando o padrão Ports & Adapters 
Versionamento: Git com estratégia GitFlow 
CI/CD: GitHub Actions para automação de testes e integração contínua 

Estrutura Arquitetural
O projeto segue o modelo de Monolito Modular, garantindo que as regras de negócio (Domínio) fiquem isoladas de detalhes técnicos como bancos de dados ou interfaces web.
Domínio: Entidades e regras centrais (Task, Sprint, User).
Aplicação: Casos de uso e orquestração do fluxo.
Infraestrutura: Implementações de banco de dados (MySQL) e segurança.
